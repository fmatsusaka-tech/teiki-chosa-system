# AI定期調査システム

まつさか農園の柑橘定期調査を、スマートフォンからできるだけ速く、簡単かつ正確に入力・確認・保存するための入力補助システムです。

決まった入力形式を覚えず、自由テキスト、スマートフォンの音声入力で作成した文章、スクリーンショット、手書きメモ写真から調査データ候補を作成できます。解析結果は人が補正してから登録します。

## 目的

- 現場での入力時間とタップ数を減らす
- 複数園地の調査結果をまとめて登録できるようにする
- 入力ミスや項目の取り違えを、登録前の確認画面で防ぐ
- 認識できない項目を捨てず、人が補正できる状態で残す

## 想定する入力方法

- 自由形式のテキスト
- スマートフォン標準の音声入力で作成した文章
- 表計算画面などのスクリーンショット
- 手書きメモの写真

入力例：

```text
徳田、早生、39.6-40.5-42.7-40.0-32.9、糖度8.4、酸度3.8
```

1つの画像や文章に複数園地が含まれる場合も、園地ごとの候補へ分割します。100%の自動認識は目指さず、確認画面での人による補正を前提にします。

## 基本的な登録の流れ

```text
入力
  ↓
AI解析
  ↓
確認・修正画面
  ↓
一括登録
  ↓
Googleスプレッドシート等への保存
```

AIの解析結果は、そのまま保存しません。必ず登録前に確認画面を表示し、利用者が修正・除外できる設計とします。

## 主な調査項目

初期段階では、次の項目を扱います。

- 計測日
- 園地
- 品種
- 横径（個数は可変）
- 糖度
- 酸度
- 備考
- 登録日時

横径は内部では配列として保持し、Googleスプレッドシートへ出力するときに「横径1、横径2……」へ展開します。

将来的には、果実数、果実重、LWP、葉色、果皮色などを追加できる構造を前提とします。

## 対象外

次の機能は別システムで行うため、本システムでは開発しません。

- 収穫時の横径・糖度・酸度予測
- 過去年度・園地・処理区の比較画面
- JAありだ／JAありだ平年値の取込と比較
- 気象データ連携

独自の音声認識も実装せず、スマートフォン標準の音声入力を利用します。

## 開発方針

- スマートフォンでの使いやすさを最優先する
- 入力形式を利用者へ強制しない
- AI解析後に必ず人が確認する
- 調査項目をハードコードせず、追加・変更しやすくする
- 内部データと外部出力形式を分離する
- 仕様が曖昧な箇所を推測だけで実装しない
- GitHubのIssueとドキュメントを仕様の基準にする

詳細な基本方針は [PROJECT_CHARTER.md](PROJECT_CHARTER.md) を参照してください。

## 開発段階

1. 自由テキストの解析
2. 解析結果の確認・修正画面
3. Googleスプレッドシートへの登録
4. スクリーンショット解析
5. 写真・手書きメモ解析
6. 実際の手書きサンプルによる認識調整
7. スマートフォン実機での受入確認と操作改善
