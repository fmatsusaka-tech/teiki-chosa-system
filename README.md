# AI定期調査システム

まつさか農園の柑橘定期調査を、スマートフォンからできるだけ速く、簡単かつ正確に入力・確認・分析するためのシステムです。

現在のGoogleフォーム中心の入力を、AIによる自由入力解析へ置き換えます。利用者は決まった入力形式を覚えず、テキスト、音声、スクリーンショット、写真などから調査データを登録できることを目指します。

## 目的

- 現場での入力時間とタップ数を減らす
- 複数園地の調査結果をまとめて登録できるようにする
- 入力ミスや項目の取り違えを、登録前の確認画面で防ぐ
- 過去年度や他園地との比較を簡単にする
- 気象データと定期調査データの関係を確認できるようにする
- 将来的に収穫時予測やストレス管理、散布判断を支援する

## 想定する入力方法

- 自由形式のテキスト
- AI音声入力・音声文字起こし
- 表計算画面などのスクリーンショット
- 手書きメモの写真
- 将来的なPDF取り込み

入力例：

```text
徳田、早生、39.6-40.5-42.7-40.0-32.9、糖度8.4、酸度3.8
```

1つの画像や文章に複数園地が含まれる場合も、AIが園地ごとのレコードへ分割します。

## 基本的な登録の流れ

```text
入力
  ↓
AI解析
  ↓
確認・修正画面
  ↓
一括登録
  ↓
Googleスプレッドシート等への保存
```

AIの解析結果は、そのまま保存しません。必ず登録前に確認画面を表示し、利用者が修正・除外できる設計とします。

## 主な調査項目

初期段階では、次の項目を扱います。

- 計測日
- 園地
- 品種
- 横径（個数は可変）
- 糖度
- 酸度
- 備考
- 登録日時

横径は内部では配列として保持し、Googleスプレッドシートへ出力するときに「横径1、横径2……」へ展開します。

将来的には、果実数、果実重、LWP、葉色、果皮色などを追加できる構造を前提とします。

## 分析機能の構想

### 定期調査分析

同じ品種・同じ調査時期の複数園地を比較します。

- 園地ごとの横径平均、糖度、酸度
- 備考の確認
- 表示する園地の選択
- グラフ比較
- 収穫時の予測値

### 園地分析

特定園地の経年推移や、2園地間の比較を行います。

- 横径、糖度、酸度の時系列
- 複数年度の重ね合わせ
- AMeDAS湯浅の降雨・気温との比較
- 積算温度・積算降雨量
- 表示項目の選択

## 開発方針

- スマートフォンでの使いやすさを最優先する
- 入力形式を利用者へ強制しない
- AI解析後に必ず人が確認する
- 調査項目をハードコードせず、追加・変更しやすくする
- 内部データと外部出力形式を分離する
- 仕様が曖昧な箇所を推測だけで実装しない
- GitHubのIssueとドキュメントを仕様の基準にする

詳細な基本方針は [PROJECT_CHARTER.md](PROJECT_CHARTER.md) を参照してください。
