# Googleスプレッドシート登録APIの設定

登録先: `定期調査データバンク` / `調査原票`

## 1. Apps Scriptを作る

1. 登録先スプレッドシートを開く。
2. `拡張機能` → `Apps Script`。
3. 初期コードを削除し、このフォルダの `Code.gs` を貼り付ける。
4. 保存する。

## 2. APIトークンを設定する

1. Apps Scriptの左側で `プロジェクトの設定` を開く。
2. `スクリプト プロパティ` に次を追加する。
   - プロパティ: `API_TOKEN`
   - 値: 推測されにくい長い文字列

## 3. Webアプリとしてデプロイする

1. `デプロイ` → `新しいデプロイ`。
2. 種類は `ウェブアプリ`。
3. 実行するユーザーは `自分`。
4. アクセスできるユーザーは `全員`。
5. デプロイ後に表示される `/exec` で終わるURLを控える。

## 4. GitHub Actionsへ登録する

リポジトリの Settings → Secrets and variables → Actions → Variables に次を追加する。

- `NEXT_PUBLIC_GAS_WEB_APP_URL`: 手順3のWebアプリURL
- `NEXT_PUBLIC_GAS_API_TOKEN`: 手順2のAPIトークン

GitHub Pagesのワークフローが環境変数を読み込むようにした後、再デプロイする。

## 登録仕様

- 同じ登録IDは再登録しない。
- 複数レコードを一度に追記する。
- 必須項目: 計測日、園地、品種、横径、糖度、酸度。
- 登録時刻はGAS側で付与する。
- 送信元の原文メモも保存する。

## 調査データの再生成

Apps Scriptエディタから `regenerateSurveyData` を実行すると、`調査原票` の全行を
`調査データ` へ再生成できます。初回実行時はGoogle Sheetsへのアクセス許可が必要です。

- `横径1`〜`横径10` は `玉1横径`〜`玉10横径` へ同じ順序でコピーします。
- 未測定の横径は空欄のまま保存し、横径の個数・平均・最小・最大の計算から除外します。
- `備考` の直後（既存の `横径個数` より前）に玉ごとの列を追加します。
- 糖度、酸度、糖酸比、データ状態、入力方法、入力者、送信元などの既存列は、見出し名で
  対応付けるため列順を変更してもずれません。予測処理向けの `横径平均` も従来どおり生成します。
- 年度（4月始まり）、年、月、調査基準月、調査区分、糖酸比、データ状態も原票から再計算します。
  25日以降は翌月の前半、1〜10日は当月の前半、11〜20日は当月の中頃として扱い、
  21〜24日の調査区分は未定義のため空欄にします。
- 糖度または酸度が欠測の場合と酸度が0の場合、糖酸比は空欄にします。調査日、園地、品種が
  すべて有効ならデータ状態は `有効`、それ以外は `要確認` とします。
