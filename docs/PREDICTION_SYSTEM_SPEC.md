# 定期調査・収穫時予測システム 仕様書

## 1. 目的

定期調査データから、既存の果樹試験場生育調査結果に基づく予測ロジックを用いて、収穫時の横径・糖度・酸度を予測する。

本システムは栽培判断を自動化するものではなく、経験者が判断しやすくするための分析補助システムとする。

## 2. システム構成

### 2.1 定期調査データの正本

Googleスプレッドシート「定期調査データバンク」の「調査データ」タブを正本とする。

- Spreadsheet ID: `1Ix7qFigeUvmxkEl3C51rmzuBzYDq7OR_ZGHq6GUKa0g`
- Sheet name: `調査データ`
- Sheet ID: `1565120965`

予測システム側では調査データを重複保存せず、正本を参照する。

### 2.2 既存予測モデル

- Spreadsheet ID: `1HntOF58ADSN_mjx3JIBxnoi3JgjoLA9zB55qRpOurKU`
- Sheets: `横径予測`、`糖度予測`、`酸度予測`

各シートの `AG:AX` にある6品種分の日付・倍数表を読み取る。

## 3. 使用項目

「調査データ」から以下を利用する。

- 登録ID
- 計測日
- 園地名
- 品種
- 処理区
- 備考
- 横径平均
- 糖度
- 酸度
- データ状態

列番号は固定せず、見出し名から列位置を解決する。

## 4. 品種と予測モデル

| 調査品種 | 予測モデル | 収穫時目標日 |
|---|---|---|
| ゆら早生 | ゆら早生 | 10月15日 |
| 早生 | 興津早生 | 11月15日 |
| 田口 | 田口早生 | 11月15日 |
| 中生 | 向山温州 | 12月1日 |
| 晩生 | 林温州 | 12月15日 |
| 丹生系 | 丹生温州 | 12月15日 |
| その他 | 予測対象外 | — |

実品種名と予測モデル名を別フィールドで保持し、将来モデルだけを差し替えられる構造とする。

## 5. 予測式

既存表と同じ倍率補正を使用する。

```text
予測値 = 現在の実測値 × 収穫時目標日の倍数 ÷ 調査日の倍数
```

予測表の3列は「日付・倍数・逆算」で構成される。計算では「倍数」を正本とし、「逆算」は派生表示として読み込まない。

横径・糖度・酸度は独立して計算する。例えば酸度が未計測でも、横径と糖度は予測する。

## 6. 予測状態

- `predicted`: 予測成功
- `not-target`: 対象外品種
- `not-measured`: 実測値なし
- `out-of-range`: モデル期間外
- `invalid-model`: モデル取得失敗または不整合

## 7. 予測結果

登録IDを関連キーとして、以下を返す。

- 調査データ一式
- 使用した予測モデル
- 横径・糖度・酸度の実測値
- 各収穫時予測値
- 調査日の倍数
- 収穫時の倍数
- 補正率
- 予測状態
- モデル版

## 8. ウェブアプリ

読み取り専用API:

```text
GET /api/predictions
```

任意の検索条件:

- `registrationId`
- `variety`
- `orchard`
- `limit`

定期調査比較画面では、園地名、処理区、備考、実測値、予測値、予測状態を一覧表示する。

## 9. スプレッドシート版

予定する表示タブ:

- 設定_品種対応
- 設定_予測モデル
- 参照_調査データ
- 予測結果
- 定期調査比較
- 園地分析

スプレッドシート版とウェブアプリ版で別の予測式を実装しない。

## 10. Google認証

アプリはGoogleサービスアカウントを使い、2つのスプレッドシートを読み取り専用で参照する。

必要な環境変数:

```text
GOOGLE_SERVICE_ACCOUNT_EMAIL
GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY
SURVEY_DATA_SPREADSHEET_ID
PREDICTION_MODEL_SPREADSHEET_ID
```

サービスアカウントのメールアドレスへ、両スプレッドシートを閲覧者として共有する必要がある。
