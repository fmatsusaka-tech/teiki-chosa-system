# 定期調査・収穫時予測システム 仕様書

## 1. 目的

定期調査データから、既存の果樹試験場生育調査結果に基づく予測ロジックを用いて、収穫時の横径・糖度・酸度を予測する。

本システムは栽培判断を自動化するものではなく、経験者が判断しやすくするための分析補助システムとする。

## 2. システム構成

### 2.1 定期調査データの正本

- Spreadsheet ID: `1Ix7qFigeUvmxkEl3C51rmzuBzYDq7OR_ZGHq6GUKa0g`
- Sheet name: `調査データ`
- Sheet ID: `1565120965`

予測システム側では調査データを重複保存せず、正本を参照する。

### 2.2 予測ロジックの正本

Googleスプレッドシート「定期検査データベース」にある以下の3シートを基礎とする。

- 横径予測
- 糖度予測
- 酸度予測

- Spreadsheet ID: `1HntOF58ADSN_mjx3JIBxnoi3JgjoLA9zB55qRpOurKU`

既存の品種別・日付別標準推移、変化係数、目標日を忠実に移植する。

## 3. 調査データ入力項目

予測計算には「調査データ」タブの以下の列を使用する。

| 項目 | 用途 |
|---|---|
| 登録ID | 一意識別子 |
| 計測日 | 予測基準日 |
| 年度 | 年度比較 |
| 調査基準月 | 調査時期検索 |
| 調査区分 | 前半・後半検索 |
| 園地名 | 園地別表示 |

| 備考 | 補助的な絞り込み |

| 酸度 | 酸度予測の現在値 |
| データ状態 | 欠測判定 |

列番号には依存せず、見出し名で列を特定する。

## 5. 予測処理

### 5.1 共通処理

1. 調査データを登録ID単位で取得する。
2. 品種対応マスタから予測モデルを決定する。
3. 対象外品種の場合は予測状態を「対象外」とする。
4. 計測日と実測値から、当日の標準値との差または比率を算出する。
5. 既存予測ロジックの品種別推移係数を用いて、収穫目標日の値を算出する。
6. 予測結果と計算根拠を表示用データとして返す。

### 5.2 欠測時の扱い

各項目は独立して予測する。

- 横径平均がある場合：横径予測を実行
- 糖度がある場合：糖度予測を実行
- 酸度がある場合：酸度予測を実行
- 未入力項目：`未計測`
- 対象外品種：`対象外`
- 係数表の期間外：`予測範囲外`

一部の項目が欠けていても、他の予測を停止しない。

## 6. 予測結果データ

予測結果は調査データの正本とは分離し、登録IDで関連付ける。

推奨項目：

| 項目 | 内容 |
|---|---|
| 登録ID | 調査データとの関連キー |
| 計算日時 | 最終計算日時 |
| 実品種 | 調査データの品種 |
| 予測モデル | 対応したモデル名 |
| 予測目標日 | 品種別目標日 |
| 横径予測値 | 収穫時横径 |
| 糖度予測値 | 収穫時糖度 |
| 酸度予測値 | 収穫時酸度 |
| 横径予測状態 | 正常・未計測・対象外・範囲外 |
| 糖度予測状態 | 正常・未計測・対象外・範囲外 |
| 酸度予測状態 | 正常・未計測・対象外・範囲外 |
| 予測ロジック版 | 再計算・検証用バージョン |

## 7. スプレッドシート版

### 7.1 役割

- 正本データを参照
- 品種対応マスタを管理
- 予測係数マスタを管理
- 予測結果一覧を表示
- 数式・計算根拠を確認
- ウェブアプリ障害時にも閲覧可能

### 7.2 推奨タブ

- `設定_品種対応`
- `設定_予測モデル`
- `参照_調査データ`
- `予測結果`
- `定期調査比較`
- `園地分析`

`参照_調査データ`は、初期段階では `IMPORTRANGE` を使用してもよい。運用負荷や処理速度が問題になった場合はGASによる同期へ切り替える。

## 8. ウェブアプリ版

### 8.1 基本方針

スプレッドシートとウェブアプリで別々の予測式を持たない。

予測処理を共通関数化し、スプレッドシート表示とウェブアプリ表示の双方から利用する。

### 8.2 定期調査比較画面

検索条件：

- 品種
- 調査基準月
- 調査区分

表示項目：

- 園地名
- 処理区
- 備考
- 横径平均
- 糖度
- 酸度
- 収穫時横径予測
- 収穫時糖度予測
- 収穫時酸度予測
- 予測状態

### 8.3 園地分析画面

検索条件：

- 園地名
- 品種
- 処理区または備考
- 年度

表示：

- 横径・糖度・酸度の時系列
- 予測値の推移
- 複数年度比較
- 将来的に湯浅AMeDASデータを重ねる

## 9. データ更新

初期実装では、画面表示時または更新操作時に予測を計算する。

安定運用後は以下を検討する。

- 調査データ追加時の自動計算
- 夜間の一括再計算
- 予測ロジック更新時の全件再計算

## 10. バージョン管理

予測係数や品種対応を変更した場合、過去結果を検証できるようにする。

- `prediction_logic_version`
- `calculated_at`
- 使用した予測モデル
- 使用した目標日

を結果に記録する。

## 11. 開発順

1. 品種対応マスタを実装
2. 既存3予測シートから予測係数を抽出
3. 横径予測関数を実装・照合
4. 糖度予測関数を実装・照合
5. 酸度予測関数を実装・照合
6. 調査データ参照処理を実装
7. 予測結果シートを実装
8. 既存スプレッドシートとテスト値を照合
9. ウェブアプリの比較画面を実装
10. 園地分析画面を実装

## 12. 完了条件

- 対応6品種で予測モデルが正しく選択される
- その他の品種は対象外になる
- 欠測項目だけが未計測になり、他項目は予測できる
- 既存予測シートと同じ入力値で、丸め誤差を除き同じ予測値になる
- スプレッドシート版とウェブアプリ版で同じ値が表示される
- 調査データ側に列追加があっても、見出し名が同じなら動作する
