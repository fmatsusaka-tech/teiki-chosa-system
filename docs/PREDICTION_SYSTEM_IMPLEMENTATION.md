# 収穫時予測機能 実装メモ

## 実装済みの範囲

- 調査品種から既存予測モデルへの対応
- 調査データを見出し名で読み取る変換処理
- 既存予測シートの `AG:AX` にある日付・倍数表の解析
- 横径・糖度・酸度を独立して予測する共通エンジン
- 日本時間基準の日付処理
- Googleサービスアカウントによる読み取り専用Sheetsクライアント
- 読み取り専用API `GET /api/predictions`
- 単体テスト

## 予測式

既存スプレッドシートと同じ倍率補正を使用する。

```text
予測値 = 現在の実測値 × 収穫時の倍数 ÷ 調査日の倍数
```

予測モデル表の「倍数」を正本とし、「逆算」は派生表示なので読み込まない。

## API

```text
GET /api/predictions
```

任意の検索条件:

- `registrationId`
- `variety`
- `orchard`
- `limit`（1〜1000、既定200）

## データ元

- 調査データ: `SURVEY_DATA_SPREADSHEET_ID` の `調査データ` タブ
- 予測モデル: `PREDICTION_MODEL_SPREADSHEET_ID` の `横径予測`、`糖度予測`、`酸度予測`

## デプロイ前に必要な設定

```text
GOOGLE_SERVICE_ACCOUNT_EMAIL
GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY
```

上記サービスアカウントのメールアドレスへ、調査データと予測モデルの2つのスプレッドシートを閲覧者として共有する。
